# Implementation Plan: Tools & MCP Integration

## 1. Overview
This plan outlines the implementation of Directive 011, which gives the agent the ability to act upon the system via local built-in skills (File System, System Command) and an MCP (Model Context Protocol) client.

## 2. Tasks

### Task 1: Setup Dependencies
- Add `mcp` to `requirements.txt`.
- Run `pip install -r requirements.txt`.

### Task 2: File System Skill
- **Test File:** `tests/skills/test_filesystem.py`
- **Source File:** `skills/local/filesystem.py`
- **Implementation:**
  - Create a sandboxed `read_file` and `write_file` skill using the `@skill` decorator.
  - Require a `sandbox_dir` constraint to prevent reading/writing arbitrary files outside the allowed scope.
  - Implement tests verifying successful reads/writes and rejecting out-of-bound paths.

### Task 3: System Command Skill
- **Test File:** `tests/skills/test_system.py`
- **Source File:** `skills/local/system.py`
- **Implementation:**
  - Create a `run_command` skill using `@skill`.
  - Use `asyncio.create_subprocess_shell`.
  - Implement timeouts and output truncation to prevent hanging or memory bloat.
  - Implement tests verifying basic execution, timeout handling, and output capture.

### Task 4: MCP Client Wrapper
- **Test File:** `tests/tools/test_mcp_client.py`
- **Source File:** `tools/mcp_client.py`
- **Implementation:**
  - Build an `MCPClientWrapper` class that connects to an MCP server using stdio transport (e.g., `mcp.client.stdio.stdio_client`).
  - Add functionality to list tools (`session.list_tools()`).
  - Add functionality to execute a tool (`session.call_tool(name, arguments)`).
  - Add a method to dynamically bind discovered MCP tools into the `SkillRegistry`.
  - Ensure telemetry (`TOOL_INVOKE`, `TOOL_COMPLETE`, `TOOL_ERROR`) is emitted around tool execution.
  - Mock the MCP session for unit tests.

### Task 5: Integration Testing
- Verify that `Agent` can actually see and invoke these tools by simulating a message to the orchestrator.
