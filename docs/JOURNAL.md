# Engineering Journal: `my-local-agent`

This journal tracks significant architectural decisions, completed directives, and bug fixes to ensure continuity across development sessions.

## 2026-03-15: Architecture Review, Tools & Telegram
**Author:** Gemini CLI

### Architecture Comparison
- Executed a deep dive into the differences between OpenClaw and our current stack. Documented findings in `TECH_STACK.md`. 
- **Conclusion:** Our stack is significantly leaner, utilizing Python and FastAPI (The "Engine" pattern) rather than OpenClaw's Node.js monolithic "Gateway" approach. This eliminates massive amounts of serialization overhead while retaining core features like Episodic Memory and Heartbeats.

### Dashboard Finalization (Directive 010)
- Built `MemoryHealth.tsx` and `HeartbeatGrid.tsx` in React using Recharts.
- Fixed an `ERESOLVE` React compilation error by forcibly installing the missing `react-is` dependency required by Recharts.
- Verified the Vite dev server hot-reloads properly and metrics flow to the frontend correctly via `useTelemetry`.

### Tools & MCP Integration (Directive 011)
- Implemented `skills/local/filesystem.py` providing `@skill(name="read_file")` and `@skill(name="write_file")` with strict sandbox directory constraints.
- Implemented `skills/local/system.py` providing an asynchronous `@skill(name="run_command")` with strict execution timeouts and output truncation to protect LLM context windows.
- Implemented `tools/mcp_client.py` using Anthropic's `mcp` library.
  - Built `MCPClientWrapper` to connect to standard `stdio` MCP servers.
  - Allowed dynamic tool discovery which binds directly to the Python `SkillRegistry`.
- Refactored `core/agent.py` to loop up to 5 times to handle `tool_calls` generated by the LLM natively.

### Channel Adapters (Discord & Telegram)
- Implemented `channels/telegram_channel.py` using `python-telegram-bot` webhooks.
- Implemented `channels/discord_channel.py` using `discord.py` client loop.
- **Bug Fix:** The `TerminalChannel` was throwing an `asyncio.exceptions.CancelledError` on shutdown because `main.py` was aggressively cancelling the core Uvicorn Server task. Rewrote `main.py` shutdown logic to respect `server.should_exit = True`, allowing a clean 5-second graceful teardown.
- **Bug Fix:** The LLM was regurgitating the entirety of `SOUL.md` to users on first interaction because it lacked separation from the rest of the system prompt. Refactored `core/agent.py` to wrap the persona rules in `<soul_instructions>` tags.
